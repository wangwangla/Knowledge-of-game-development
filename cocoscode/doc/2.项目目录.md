# 项目目录

都放入一个resource会将所有的文件都会进行加载 ，asset资源

scene：场景文件 

script：保存脚本

每一个文件夹都会生成一个mate的，管理文件会生成mate，生成的uuid来管理文件夹下的资源。

一般都可以删除掉library  local  package  setting文件夹 ，当在次打开的时候又会重新生成。

## 组件

都是需要继承cc.componet的。

properties：属性，可以编译完之后会绑定到编译器上的。

组件入口方法：

- onLoad：加载的时候使用
- start:开始的时候使用
- update:游戏每次更新的时候
- onEnable:可以点击
- onDisable:不可以点击的时候

打印函数：console.log();

## cc.Node 

#### cc.node的属性

- name:获取节点的名字
- active:设置可见性

- rotation:旋转，顺时针为正，逆时针为负
- scale:缩放
- anchor:锚点，00  ->  11
- size:大小
- color:颜色光
- opacity:透明度
- skew:扭曲
- group：分组
- parent/child
- tag

## 组件

- 都是继承自cc.Component（类 ）

- 存在一个Node,指向它关联的节点cc.Node

- 可以通过name获取到节点的名称

- 组件实例的入口函数
  - onLoad：加载组件的时候调用
  - start:组件第一次激活前，调用载第一次update之前
  - update：每次游戏刷新都会执行
  - lateUpdate:在update之后执行
  - enable:组件是否被启用
  - onEnable:组件被允许的时候调用
  - onDisable:组件不被允许的时候调用

组件实例：

- 我们将一个组件 挂在多次就会创建多个实例，可以打印this进行查看
